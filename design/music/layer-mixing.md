# 《暗室》层级混音（design/music/layer-mixing.md）

> **一句话定位：** 5 层级架构（主旋律 / 伴奏 / 节奏 / 氛围 / 效果）+ 4 强度反馈分层 + 5 动态混音规则 + 19 房间混音参数 + 4×9 混音矩阵 + 心流曲线 7 阶段。

## 目的 (Purpose)

本文档是《暗室》音乐层**层级混音架构**的完整定义。它向：

- **音乐总监 / 音频工程师** — 讲清**5 层级架构**（主旋律 / 伴奏 / 节奏 / 氛围 / 效果）+ **4 强度反馈对应分层** + **5 动态混音规则**
- **关卡策划** — 19 房间每房间**5 层级独立音量参数**（与房间主题氛围对应）
- **Unity 工程师** — `LayerMixController.cs` 模块的 5 AudioMixer Group 配置
- **外包商** — **4 强度反馈 → 9 类音频混合矩阵**的明确契约
- **QA / 测试** — 心流曲线 7 阶段音频对应 + 验收基线

**本文档范围：** **混音（Mixing）+ 层级（Layering）**——音量分层、动态混音规则、心流曲线；不涉及乐器 / 编码 / 平台（详见 `instruments.md` + `standards.md` + `cross-platform.md`）。

## 1. 5 层级架构 (5-Layer Architecture)

> **核心设计原则：** **5 层级**（主旋律 / 伴奏 / 节奏 / 氛围 / 效果）——每个层级**独立音量控制** + **独立触发条件** + **独立乐器清单**——保证"分章节混音"和"分强度反馈混音"。

### 1.1 5 层级架构总览 (5-Layer Architecture)

| 层 | 名称 | 用途 | 典型乐器 | 音量范围 | 4 Pillar 对应 | AudioMixer Group |
|---|------|------|---------|---------|--------------|------------------|
| **L1** | **主旋律层** (Melody) | 章节 BGM 主旋律 + 通关音主旋律 | 大提琴 / 钢琴 / 弦乐合奏 / 钟声 | -9 ~ 0 dB | P4 主题回归 | BGM_Melody |
| **L2** | **伴奏层** (Accompaniment) | 章节 BGM 伴奏 + 主旋律变奏 | 小提琴 / 长笛 / 竖琴 / 合成器 | -12 ~ -6 dB | P1 沉浸感 | BGM_Accompaniment |
| **L3** | **节奏层** (Rhythm) | 章节 BGM 节奏 + 鼓点 + 沙锤 | 手鼓 / 沙锤 / 钢片琴 | -15 ~ -9 dB | P2 拓扑重构 | BGM_Rhythm |
| **L4** | **氛围层** (Ambience) | 室内 / 走廊 / 电流声 + 环境音 | pad / 风声 / 远处水滴 | -18 ~ -12 dB | P1 沉浸感 | Ambient |
| **L5** | **效果层** (Effects) | 切换 / 重置 / 错音 / 通关 / UI 反馈 | 切换音 / 重置音 / 错音 / UI 音 | -18 ~ -3 dB | P2 拓扑重构 + P3 视觉欺骗 | SFX + UI |

> **关键设计：** 5 层级**音量范围不重叠**——主旋律 -9 ~ 0dB 最高，效果层 -18 ~ -3dB 跨度大，节奏层 -15 ~ -9dB 中等，避免"层级间音量冲突"。

### 1.2 5 层级与 4 Pillar 映射 (5 Layers × 4 Pillars)

```
Pillar 1 沉浸感 (dark ambient)        → L2 伴奏层（极远声场）
                                    → L4 氛围层（环境音 + 风声）

Pillar 2 拓扑重构惊喜 (puzzle)       → L3 节奏层（鼓点 + 不和谐音）
                                    → L5 效果层（切换音 + 重置音）

Pillar 3 视觉欺骗 (minimalist)       → L1 主旋律层（极简 5 音符）
                                    → L5 效果层（错音 + 静音突变）

Pillar 4 一致性 (主题循环)           → L1 主旋律层（90s 循环 + 通关回归）
                                    → L2 伴奏层（贯穿 19 房间）
```

### 1.3 5 层级与 4 强度反馈映射 (5 Layers × 4 Intensity)

| 层 | L1 轻反馈 | L2 中反馈 | L3 重反馈 | L4 兜底 |
|---|----------|----------|----------|--------|
| **L1 主旋律** | -18 dB | -12 dB | -6 dB | 0 dB |
| **L2 伴奏** | -18 dB | -12 dB | -6 dB | -3 dB（同时加 + 18dB 节奏层）|
| **L3 节奏** | -18 dB | -12 dB | -6 dB | **+18 dB（最强）**（"我卡住了" 暗示）|
| **L4 氛围** | -18 dB | -15 dB | -12 dB | **-25 dB（最弱，逼近静音）**|
| **L5 效果** | -12 dB（切换音） | -12 dB（错音）| -12 dB ×2（错音连发）| **静音 → 5s 后风铃 -12dB** |

> **关键设计：** L4 兜底时**氛围层逼近静音 + 节奏层 +18dB**——"最大警告"通过"环境安静 + 节奏加大"双重表达，而非"突然大声"。

## 2. 4 强度反馈对应分层 (4 Intensity Layer Mapping)

> **设计原则：** 4 强度反馈**音量递增 + 风格递增**——传达"问题越来越严重但游戏不惩罚你"（与 07-v2 §7 + 09-v2 §5.1 + style.md §5 一致）。

### 2.1 4 强度反馈音量分层表 (4 Intensity Volume Layer Table)

| 强度 | L1 主旋律 | L2 伴奏 | L3 节奏 | L4 氛围 | L5 效果 | 触发场景 |
|------|:--------:|:------:|:------:|:------:|:------:|---------|
| **L1 轻反馈** | -18 dB | -18 dB | -18 dB | -18 dB | -12 dB | F-α 软失败（1 次切换错）|
| **L2 中反馈** | -12 dB | -12 dB | -12 dB | -15 dB | -12 dB | F-β 硬失败（3 次错误配置）|
| **L3 重反馈** | -6 dB | -6 dB | -6 dB | -12 dB | -12 dB ×2 | F-β 持续（5+ 次错误）|
| **L4 兜底** | 0 dB | -3 dB | **+18 dB** | **-25 dB** | 静音 → -12 dB 风铃 | F-γ 卡住（30+ min）|

> **关键设计：** 4 强度**音量递增**（-18 → -12 → -6 → 0）——主旋律**逐级上升**，强化"问题严重"。
> **关键设计：** L4 兜底**节奏 +18dB** 是**最大警告**——但因环境 -25dB 逼近静音，整体音量**反而平衡**（不爆音）。

### 2.2 4 强度反馈层级触发契约 (4 Intensity Trigger Contract)

```
玩家错误次数 1 次  → L1 轻反馈（L1 主旋律 -18dB + L5 切换音 -12dB）
玩家错误次数 3 次  → L2 中反馈（L1 主旋律 -12dB + L5 错音 -12dB）
玩家错误次数 5+ 次 → L3 重反馈（L1 主旋律 -6dB + L5 错音 ×2 -12dB）
玩家停留 30+ min  → L4 兜底（L1 主旋律 0dB + L4 氛围 -25dB + L3 节奏 +18dB + L5 静音 → 5s 风铃）
```

> **实现：** `src/Audio/IntensityFeedbackAudio.cs` 模块读取 `PlayerPrefs.errorCount` + `TimeInRoom`，触发对应强度混合模式。

## 3. 5 动态混音规则 (5 Dynamic Mixing Rules)

> **设计原则：** 5 动态混音规则**全部参数响应**——音频在不同玩家位置 / 章节 / 难度下**音量不同**（与 09-v2 §2 一致）。

### 3.1 5 动态混音规则清单 (5 Dynamic Mixing Rules)

| # | 规则 | 触发条件 | 参数响应 | 来源 |
|---|------|---------|---------|------|
| **DM-01** | **玩家距 SwitchSlot 渐强规则** | 玩家距 SwitchSlot 距离（格数）| L4 氛围电流声音量 -18 → -6dB（距 0 格最响）| 09-v2 §2.1 |
| **DM-02** | **章节 BGM 切换规则** | 章节过渡 / 通关 | L1+L2 渐出 + L1+L2 渐入（2-3s cross-fade）| 09-v2 §2.2 |
| **DM-03** | **难度递增动态混音** | 玩家进入高难度房间 | L3 节奏层从 0% 升至 100% 密度 | 09-v2 §2.3 + adaptive §7 |
| **DM-04** | **房间进入混音规则** | 玩家进入新房间 | L2 伴奏变奏（章节 BGM 不同循环点 + 房间主题）| 09-v2 §2.4 |
| **DM-05** | **反馈 3 层混音时序契约** | 玩家按 E / R / Q | L5 效果层 attack ≤ 50ms / decay ≤ 200ms（与视觉同步）| 09-v2 §2.5 + 08-v2 §7.3 |

> **关键设计：** 5 规则**按 Pillar 分配**——DM-01/DM-02 对应 P1 沉浸感（dark ambient），DM-03/DM-04 对应 P4 一致性（主题循环），DM-05 对应 P2 拓扑重构（puzzle 突变）。

### 3.2 DM-01: 玩家距 SwitchSlot 渐强规则

> **源约束：** 06-v2 §11.1.2 沉浸感细节"动态音量 距槽位 1 格渐强"

```
玩家距 SwitchSlot 距离（格数） → L4 氛围电流声音量（dB）
距 0 格（玩家站在槽位上）     → -6 dB（最大）
距 1 格                       → -12 dB（中）
距 2 格                       → -18 dB（最小可闻）
距 3+ 格                      → 静音
```

**实现：** 每帧检测玩家与最近 SwitchSlot 距离，线性插值 -18dB → -6dB（距 2 格 → 距 0 格）。详见 `adaptive-system.md` §1 + `src/Audio/AmbientAudioSystem.cs`。

### 3.3 DM-02: 章节 BGM 切换规则

| 触发点 | L1 主旋律 + L2 伴奏动作 | L1+L2 切换时长 | 引用 |
|-------|-----------------------|---------------|------|
| **BootUp → MainMenu** | 加载 `bgm_mainmenu_01`（L1 + L2 同步）| 0s（直接播放）| 09-v2 §2.2 |
| **MainMenu → ChapterSelect → RoomEntry** | 章节 BGM 渐入（L1+L2）| 2s fade-in | 09-v2 §2.2 |
| **章节通关 → 章节标题画面** | 章节 BGM 继续 + L2 章节提示音叠加（1s）| 0s（叠播）| 09-v2 §2.2 |
| **末章通关** | L1+L2 渐出 + 通关 BGM 渐入 | 3s cross-fade | 09-v2 §2.2 |
| **通关 → 制作名单** | L1+L2 渐出 | 2s fade-out | 09-v2 §2.2 |

### 3.4 DM-03: 难度递增动态混音

> **P0-001 跟踪：** 02-v2 §13 AC-06 仍缺"难度上限 20"硬约束。本规则依赖该约束——**待 P0-001 解决后生效**。

| 房间难度 | L3 节奏层密度 | 引用 |
|---------|:-----------:|------|
| 1-5（难度 2-5）| **0%**（仅章节 BGM 基础层）| 09-v2 §2.3 |
| 2-1 ~ 2-6（难度 7-10）| **50%**（L3 节奏层进入，密度 50%）| 09-v2 §2.3 |
| 3-1 ~ 3-7（难度 11-16）| **100%**（L3 节奏层全密度）| 09-v2 §2.3 |
| 3-8（难度 16，Boss 房）| **100% + Boss 房专属 BGM** | 09-v2 §2.3 + §1.8 |

> **实现：** L3 节奏层密度 = 多个 stem 同步播放（基础层 / 节奏层 / 紧张层），各 stem 音量独立控制。**v1.0 简化版：仅章节 BGM 切换，不做 stem 动态混音**。
> **P0-001 影响：** 若 02-v2 AC-06 增补"难度上限 20"，本规则表可从 4 档简化为 3 档（去掉 Boss 房独立档，归入难度 11-16 区间）。

### 3.5 DM-04: 房间进入混音规则

| **房间内移动** | 章节 BGM 持续，无变化 | — | 09-v2 §2.4 |

| **章节末间通关** | 章节 BGM 持续 + L2 章节提示音叠加（1s）| 0s（叠播 1s 后淡出）| 09-v2 §2.4 |

### 3.6 DM-05: 反馈 3 层混音时序契约

> **源约束：** 08-ui-ux-v2.md §7.1 3 层反馈定义 + 02 §7.2 切换时序

```
E 键按下 t=0ms     → L5 效果层（16ms）+ L5 视觉同步（200ms）+ L5 触觉同步（16ms）三通道同步启动
L5 效果层 0-50ms    → 切换音快速 attack（避免爆音）
L5 效果层 50-200ms  → 切换音 decay 至静音
L1 主旋律 0-200ms   → 槽位发光增强 + 旧预制件淡出
L5 触觉 0-16ms      → 手柄震动（仅手柄）
t=200ms 切换完成    → 玩家位置 / 状态回归 Hover
```

> **关键设计：** L5 效果层 attack 必须 ≤ 50ms（防爆音），decay ≤ 200ms（与切换动画同步结束）。

## 5. 4 强度反馈 → 9 类音频混合矩阵 (4 Intensity × 9 Categories)

> **设计原则：** 9 类音频在 L1-L4 时**音量变化表**——明确"哪类音频在哪种强度下应该多响 / 少响"。

### 5.1 4 × 9 混合矩阵 (4 × 9 Mix Matrix)

| 9 类音频 | L1 轻反馈 | L2 中反馈 | L3 重反馈 | L4 兜底 | 风格层 |
|---------|:--------:|:--------:|:--------:|:--------:|--------|
| **A1 切换音** | -12 dB（短促 1 次）| -12 dB（短促 1 次）| -12 dB（短促 1 次）| **静音** | L5 效果 |
| **A2 重置音** | -18 dB（短促 1 次）| -18 dB（短促 1 次）| -18 dB（短促 1 次）| **静音** | L5 效果 |
| **A3 通关音** | 0 dB（触发时）| 0 dB（触发时）| 0 dB（触发时）| 0 dB（触发时）| L1 主旋律 |
| **A4 错音** | -18 dB（不触发）| -12 dB（短促 1 次）| **-12 dB ×2**（连发）| **静音** | L5 效果 |
| **A5 教学音** | -12 dB（首次教学）| -12 dB（首次教学）| -12 dB（首次教学）| -12 dB（首次教学）| L1 主旋律 |
| **A6 章节 BGM** | -6 dB（持续）| -6 dB（持续）| -6 dB（持续）| **-25 dB（逼近静音）**| L1 主旋律 |
| **A7 房间主题** | -6 dB（持续）| -6 dB（持续）| -6 dB（持续）| **-25 dB（逼近静音）**| L1+L2 |
| **A8 环境音** | -18 dB（室内）/ -15 dB（走廊）/ -6dB（电流，靠近时）| 同 L1 | 同 L1 | **静音** | L4 氛围 |
| **A9 UI 反馈音** | -12 dB / -15 dB / -9 dB | 同 L1 | 同 L1 | **-12 dB 风铃（5s 后）** | L5 效果 |

> **关键设计：** A3 通关音**不受强度反馈影响**——通关音始终 0dB / -3dB（最高优先级），强化"通关 = 主要成就"。
> **关键设计：** L4 兜底时**A6+A7 BGM 逼近静音**——"我卡住了" 暗示，配合"全房间光线变暗"形成"无助感"。

### 5.2 4×9 混合矩阵与 5 层级对应 (Matrix × 5 Layers)

| 9 类音频 | 5 层级映射 | 4 强度反馈音量变化 |
|---------|-----------|------------------|
| A1 切换音 | L5 效果 | -12dB → -12dB → -12dB → 静音 |
| A2 重置音 | L5 效果 | -18dB → -18dB → -18dB → 静音 |
| A3 通关音 | L1 主旋律 | 0dB（始终不变）|
| A4 错音 | L5 效果 | -18 → -12dB → -12dB ×2 → 静音 |
| A5 教学音 | L1 主旋律 | -12dB（始终不变，仅触发 1 次）|
| A6 章节 BGM | L1 主旋律 | -6dB → -6dB → -6dB → -25dB |
| A7 房间主题 | L1+L2 | -6dB → -6dB → -6dB → -25dB |
| A8 环境音 | L4 氛围 | -18dB/-15dB/-6dB（距槽位渐强）→ 静音 |
| A9 UI 反馈音 | L5 效果 | -12/-15/-9dB → -12dB 风铃 |

## 6. 心流曲线 (Flow Curve)

> **设计原则：** 心流（Flow）= Csikszentmihalyi 模型——**挑战与技能匹配**——7 阶段音频对应**与玩家状态匹配**（与 06-v2 §6 心流 + 09-v2 §2 动态混音一致）。

### 6.1 7 阶段心流曲线 (Csikszentmihalyi 7-Phase Flow)

| 阶段 | 名称 | 玩家状态 | 音频对应 | L1 主旋律 | L2 伴奏 | L3 节奏 | L4 氛围 | L5 效果 |
|------|------|---------|---------|:--------:|:------:|:------:|:------:|:------:|
| **S1** | **起始** | 玩家进入游戏 | 主菜单 BGM | -6 dB | -12 dB | -18 dB | -18 dB | -12 dB |
| **S2** | **探索** | 玩家探索新房间 | 章节 BGM 基础层 | -6 dB | -12 dB | -18 dB | -18 dB | -12 dB |
| **S3** | **挑战** | 玩家进入难房 | 章节 BGM 基础 + 节奏层（50%）| -6 dB | -12 dB | -12 dB | -18 dB | -12 dB |
| **S4** | **挫败** | 玩家多次错误 | L4 强度反馈（L1-L2 触发）+ 静音错音 | -12 dB | -12 dB | -12 dB | -15 dB | -12 dB |
| **S5** | **突破** | 玩家即将通关 | 章节 BGM 节奏层加密 + 短促通关音预告 | -3 dB | -9 dB | -6 dB | -18 dB | -6 dB |
| **S6** | **掌控** | 玩家通关房间 | 主旋律回归（片段）| -3 dB | -9 dB | -12 dB | -18 dB | -3 dB |
| **S7** | **完成** | 玩家通关全章 / 通关游戏 | 主旋律完整回归 + 全乐器 ff + 钟声 | 0 dB | -3 dB | -3 dB | -18 dB | -3 dB |

> **关键设计：** 7 阶段**音量逐级上升**（S1 -6dB → S7 0dB）——强化"我从挑战走到掌控"。
> **关键设计：** S4 挫败**用 L4 强度反馈**触发——而非"突然大声"——传达"游戏不惩罚你"。

### 6.2 心流曲线与 4 强度反馈交叉 (Flow × Intensity)

```
玩家错误数 1 次  → S2 探索（基础 + L1）
玩家错误数 3 次  → S3 挑战 → S4 挫败（基础 + L2 错音）
玩家错误数 5+ 次 → S4 挫败（基础 + L3 错音 ×2）
玩家停留 30+ min → S4 挫败（基础 + L4 兜底 -25dB/+18dB）
玩家通关房间    → S5 突破 → S6 掌控（主旋律回归片段）
玩家通关章节    → S7 完成（主旋律回归完整）
玩家通关游戏    → S7 完成（主旋律完整 + 全乐器 + 钟声）
```

> **实现：** `src/Audio/AdaptiveAudioDirector.cs` 模块读取 `PlayerPrefs.errorCount` + `TimeInRoom` + `currentRoomId` + `gameState`，触发对应心流阶段音频参数。

## 7. 边界条件 (Edge Cases)

1. **玩家快速连按切换键（每秒 10 次）**
   - **触发条件：** 玩家测试或焦虑时连按 E/Q
   - **音频预期：** 300ms 内只播放 1 次 L5 效果层切换音（-12dB），其余按键**不播放音频**

2. **章节 BGM 切换时玩家正在通关**
   - **触发条件：** 玩家在章节末间通关瞬间 + 章节 BGM 切换瞬间重叠
   - **音频预期：** L1+L2 章节 BGM 不立即切走，先播放 L3 通关音（-6dB / -3dB），通关音结束后再切下章 BGM

3. **Boss 房 BGM 与章节 BGM 风格冲突**
   - **触发条件：** 玩家进入 3-8 Boss 房时 L1+L2 渐出 + Boss 房 BGM 渐入
   - **音频预期：** Boss 房 BGM 用**同主旋律的紧迫变奏**（非独立 BGM），3s cross-fade 避免断裂

4. **房间 BGM 变奏失败（同 BGM 不同循环点对不上）**
   - **触发条件：** 玩家通关后从 1-2 → 1-3，BGM 循环点对不上
   - **音频预期：** L1+L2 重新从循环点（84s / 86s / 88s）开始播放，1s fade-in 避免断裂

5. **低帧率 30 FPS 时 5 层级混音线程阻塞**
   - **触发条件：** 低端机 CPU 占用 > 80%
   - **音频预期：** 5 层级混音**不依赖帧率**（独立线程），即使帧率掉到 30 FPS 音频依然流畅

6. **L4 兜底时玩家突然通关**
   - **触发条件：** 玩家停留 30+ min 触发 L4 兜底后 5s 内通关
   - **音频预期：** L4 兜底被 L3 通关音（-6dB / -3dB）打断，主旋律回归立即触发
