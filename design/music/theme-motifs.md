# 《暗室》主题与动机（design/music/theme-motifs.md）

> **一句话定位：** SwitchSlot 切换是核心隐喻 → "转换 / 觉醒 / 连接"三主题 → 19 房间主题映射 → 5 段落音色 → 通关主旋律回归 → 5 应急主题。

## 目的 (Purpose)

本文档是《暗室》音乐层**主题（Theme）与动机（Motif）**的完整设计。它向：

- **音乐总监 / 音频工程师** — 讲清**主线剧情主题** + **19 房间主题** + **5 段落音色** + **通关回归**
- **关卡策划** — 每房间主题与 03-v2 房间配置的对应关系 + 4 强度反馈触发
- **Unity 工程师** — ChapterProgressionMixer + RoomThemePlayer 模块的动机数据
- **外包商** — 5 应急主题（EP-1 ~ EP-5）的触发条件与替换策略

**本文档范围：** 主题（旋律 / 和声 / 节奏型）+ 动机（主旋律片段）+ 变奏（重复 + 变化）。

## 1. 主线剧情主题 (Main Plot Themes)

> **核心设计原则：** SwitchSlot 切换是《暗室》核心隐喻——玩家通过**切换**改变拓扑 / **觉醒**发现真相 / **连接**所有房间。音乐主题**强化隐喻**。

### 1.1 三大主线主题 (3 Main Themes)

| 主题 | 哲学意涵 | 音乐实现 | 关联机制 | 在 5 段落音色中的位置 |
|------|---------|---------|---------|---------------------|
| **转换 (Transformation)** | "切换 = 重构现实" | 短促切分音 + 半音下行（C5→B4）| 02-v2 §3 SwitchSlot 切换 | 每段主旋律**核心动机** |
| **觉醒 (Awakening)** | "认知改变 = 顿悟时刻" | 大三和弦解决 + 上升音阶（C5→G5→C6）| 02-v2 §7.2 切换时序（200ms）| 通关音 + 主菜单回归 |
| **连接 (Connection)** | "所有房间构成网络" | 循环音型 + 渐进和声（i → IV → V → i）| 02-v2 §11.3 拓扑连通判定 | 章节 BGM 基础层 |

> **关键设计：** 三大主题**不是顺序出现**，而是**交织**——切换音里含"觉醒"动机，章节 BGM 里含"连接"动机，强化"我与世界是一个整体"的核心体验。

### 1.2 主旋律原型 (Main Melody Prototype)

> **设计原则：** 主旋律**只用 5 个音符**（C-D-E-G-A = 大调五声音阶）——无半音 / 无复杂节奏，符合"极简"风格 + "无小节切分"。

```
主线动机（M1 = 转换动机）：
   C5 (1/4) → D5 (1/8) → E5 (1/4) → G5 (1/2) → E5 (1/4) → D5 (1/4)
   ↓ (通關音)
   C5 (1/4) → G5 (1/2) → E5 (1/4) → D5 (1/8) → C5 (1/2)
   
   8 小節一组，反復時可變奏為：
   - V1: 8 分音符 → 16 分音符（加密）
   - V2: 純和聲（弦樂墊底 + 主旋律簡化）
   - V3: 高 8 度重複（鋼琴 → 鋼琴高音）
```

### 1.3 主旋律情绪变化 (Melody Emotional Arc)

```
位置           情绪        力度 (dynamics)   速度 (bpm)   乐器
主菜单         mysterious  pp (pianissimo)  60 bpm      大提琴 + pad
Ch1 起始       calm        p (piano)         72 bpm      钢琴 + 弦乐
Ch1 5 房间末尾  rising      mf (mezzo-forte) 80 bpm      钢琴 + 弦乐 + 风
Ch2 起始        tense       p                 72 bpm      合成器 + 低频
Ch2 5 房间末尾  rising      mf                80 bpm      合成器 + 低频 + 鼓
Ch3 起始        mysterious  mp (mezzo-piano) 80 bpm      合成器 + 鼓点
Ch3 7 房间末尾  tense       f (forte)         90 bpm      全乐器
Ch3 8 Boss 房   intense     ff (fortissimo)   100 bpm     全乐器 + 不和谐
通关 (3-8)     relief       mf→p              60 bpm      钢琴 + 弦乐 + 钟声（回归主旋律）
```

## 3. SwitchSlot 切换动机 (SwitchSlot Switching Motif)

> **核心设计：** 4 槽位类型对应 4 主题家族——**每个槽位类型有专属动机**，与主题家族强耦合。

### 3.1 4 槽位类型对应 4 主题家族 (4 Slot Types × 4 Theme Families)

| 槽位类型 | 主题家族 | 动机编号 | 动机描述 | 音量 dB |
|---------|---------|---------|---------|---------|
| **ToggleSlot (TS)** | **物理 (Physics)** | M-PHY | 短促切分音 + 金属质感（钹 / 钢片琴）| -12 |
| **CycleSlot (CS)** | **逻辑 (Logic)** | M-LOG | 连续 3 咔哒（0.07s 间隔）+ 上升音 | -12 |
| **ConditionalSlot (CDS)** | **感知 (Perception)** | M-PER | 通电声（150-300Hz 上升）+ 弦乐拨奏 | -12 |
| **LockedSlot (LS)** | **记忆 (Memory)** | M-MEM | 金属拒绝声 + 低沉鼓（80-150Hz）| -18（错音）|

> **设计哲学：** 4 主题家族（物理 / 逻辑 / 感知 / 记忆）对应玩家**4 种认知能力**——物理（空间感）、逻辑（推理）、感知（视觉）、记忆（经验）。SwitchSlot 4 类型**强化**玩家"我在用某种能力解决"的元认知。

### 3.2 切换动机播放契约 (Switching Motif Playback Contract)

```
玩家按 E 键 (t=0)
    ↓
[0-50ms]  AudioManager.PlaySFX(SwitchMotif, dB=-12)
          ↓
[50-200ms] AudioManager.PlaySFX(Decay, dB=-12 → -inf)
             ↓
[200ms]    切换完成 + 旧预制件淡出 + 新预制件淡入（与 08-v2 §7 反馈 3 层同步）
```

> **关键设计：** 切换动机与 08-v2 §7.3 反馈 3 层契约**精确同步**——音频 attack ≤ 50ms，decay ≤ 200ms（防爆音 + 与视觉同步结束）。

## 4. 主题变奏 (5 段落音色：1 主旋律 + 3 变奏)

> **设计原则：** **5 段落音色**（序章 / 第一章 / 间章 / 第二章 / 第三章），每段**1 主旋律 + 3 变奏**——v1.0 仅主旋律实施（v1.1 评估 3 变奏展开）。

### 4.1 5 段落音色总览 (5-Section Timbres Overview)

| 段落 | 章节 | 主旋律乐器 | 风格标签 | 时长 / 循环点 | 音量 dB | 3 变奏（v1.1 评估）|
|------|------|----------|---------|------|---------|------------------|
| **序章** | 主菜单 | 大提琴 | mysterious / calm | 60s / 56s | -6 | V1 简化 / V2 加密 / V3 高 8 度 |
| **第一章** | Ch1 觉醒 | 钢琴 + 弦乐 | calm / explore | 90s / 84s | -6 | V1 + 风 / V2 + 鼓 / V3 全乐器 |
| **间章** | 章节过渡 | 风铃 + pad | transition / neutral | 8s / 无循环 | -9 | V1 大提琴 / V2 钟声 / V3 通关音 |
| **第二章** | Ch2 深掘 | 合成器 + 低频 | tense / curious | 90s / 86s | -6 | V1 + 鼓 / V2 + 不和谐 / V3 全乐器 |
| **第三章** | Ch3 迷途 | 合成器 + 鼓点 + 突变音 | mystery / rhythm | 90s / 88s | -6 | V1 + 不和谐 / V2 高频 / V3 Boss 房 |

> **循环点说明：** 章节 BGM 60-90s 内有**自然循环点**（如主旋律结束时），让 90s 循环不出现"断裂感"。
> **关键设计：** v1.0 **仅主旋律**实施——3 变奏推到 v1.1（与 09-v2 §13 Q-02 一致）。

### 4.2 主旋律变奏规则 (Main Melody Variation Rules)

```
主旋律 (V0 = 原版)：
- 8 小節，C 大調五声音階 (C-D-E-G-A)
- 鋼琴主奏 + 弦樂衬托
- bpm 72，力度 mp

V1 简化变奏（v1.1）：
- 4 小節，去除裝飾音
- 大提琴独奏 + pad
- bpm 60，力度 pp

V2 加密变奏（v1.1）：
- 16 小節，加入 16 分音符 + 颤音
- 鋼琴 + 弦樂 + 合成器
- bpm 90，力度 mf

V3 高 8 度变奏（v1.1）：
- 8 小節，移調 8 度（C6-D6-E6-G6-A6）
- 鋼琴高音独奏 + 钟聲
- bpm 80，力度 f
```

### 4.3 5 段落过渡契约 (5-Section Transition Contract)

| 过渡点 | 切换动作 | 切换时长 | 引用 |
|-------|---------|---------|------|
| **BootUp → MainMenu** | 加载 `bgm_mainmenu_01` | 0s（直接播放）| 09-v2 §2.2 |
| **MainMenu → ChapterSelect → RoomEntry** | 当前章节 BGM 渐入 | 2s fade-in | 09-v2 §2.2 |
| **ChapterComplete → Interlude** | 章节 BGM 渐出 + 间章音渐入 | 3s cross-fade | 09-v2 §2.2 |
| **Interlude → NextChapter** | 间章音渐出 + 下章 BGM 渐入 | 2s cross-fade | 09-v2 §2.2 |
| **末章通关** | 章节 BGM 渐出 + 通关 BGM 渐入 | 3s cross-fade | 09-v2 §2.2 |

> **关键设计：** 章节 BGM **不打断**，仅 fade-in/fade-out（2-3s），避免"突然静音"或"突然大声"。

## 5. 主题回归 (Main Theme Recurrence)

> **设计原则：** 通关时**主旋律回归**——玩家完成度触发，强化"我从转换走到觉醒"的旅程。

### 5.1 主旋律回归触发条件 (Recurrence Triggers)

| 触发点 | 回归动作 | 触发条件 | 音量 dB | 引用 |
|-------|---------|---------|---------|------|
| **通关 (3-8)** | 主旋律完整回归 + 弦乐合奏 + 钟声 | 玩家通关末间 | -3（最高）| 09-v2 §1.4 A3 通关音 + §3.1 |
| **章节通关 (1-5 / 2-6)** | 主旋律简化回归 + 章节提示音叠加 | 章节末间通关 | -3 | 09-v2 §1.4 A3 |
| **房间通关（普通房）** | 主旋律片段（2 小节）+ 通关音 | 房间通关 | -6 | 09-v2 §1.4 A3 |
| **Boss 房 (3-7) 通关** | 主旋律片段 + Boss 房 BGM 渐出 | Boss 房通关 | -6 | 09-v2 §1.7 A7 |
| **玩家重玩已通关房** | 章节 BGM 持续，无主旋律回归 | 玩家选已通关房 | -6 | 不回归（避免疲劳）|

> **关键设计：** 主题回归**仅在玩家首次通关时触发**——重玩已通关房 / 房间**不回归**，避免"回归疲劳"。

### 5.2 主旋律回归音效设计 (Recurrence Audio Design)

```
通关 (3-8) 主旋律回归：
t=0ms     → 通关音 -6dB 触发（sfx_win_room_01）
t=500ms   → 主旋律回归（M1 动机 + 大提琴 + 钢琴 + 弦乐合奏）
t=8000ms  → 通关 BGM 完整淡入（bgm_ending_01，30s 一次性）
t=10000ms → 制作名单滚动静音
```

## 6. 5 应急主题 (5 Emergency-Plan Themes)

> **设计原则：** 5 应急主题（EP-1 ~ EP-5）覆盖**极端情况**——Steam 审核不通过 / 试玩版无反馈 / KOL 关注度低 / P0-001 阻塞 / Itch.io 失败——为后续版本切换预留主题。

### 6.1 5 应急主题清单 (5 EP Themes)

| EP 编号 | 应急场景 | 主题内容 | 触发条件 | 替换策略 |
|--------|---------|---------|---------|---------|
| **EP-1** | **Steam 审核不通过** | "缓和主题"（低强度 + 缓和情绪）| 审核要求降低情绪强度 | 主旋律替换为 EP-1 版本（5h 工时）|
| **EP-2** | **试玩版无反馈** | "新手指引主题"（明亮 + 渐强）| Itch.io 试玩版需要新手指引音乐 | Ch1 BGM 替换为 EP-2 版本（3h 工时）|
| **EP-3** | **KOL 关注度低** | "戏剧化主题"（紧迫 + 戏剧反差）| KOL 反馈需要更强戏剧张力 | 章节 BGM 替换为 EP-3 版本（8h 工时）|
| **EP-4** | **P0-001 阻塞** | "稳定主题"（无难度跟随音频）| P0-001 未解决，无需难度跟随音频 | adaptive-system §7 难度自适应**禁用**（0h 工时）|
| **EP-5** | **Itch.io 失败** | "紧缩主题"（减少循环点 + 短时长）| Itch.io 试玩版时长限制需要短 BGM | 主菜单 BGM 替换为 EP-5 版本（2h 工时）|

> **关键设计：** 5 EP **都不是预先制作**——仅在应急触发时**后续制作**，避免 v1.0 投入过量制作资源（符合 1 人 Solo + $0-30/月 预算）。

|---------|------|---------|
| EP 主题渐入 | 3s fade-in | -6 |

## 7. 边界条件 (Edge Cases)

1. **玩家快速连按切换键（每秒 10 次）**
   - **触发条件：** 玩家测试或焦虑时连按 E/Q
   - **音频预期：** 300ms 内只播放 1 次切换音（-12dB），其余按键**不播放音频**（与 02 §10.3 冷却同步）

2. **章节 BGM 切换时玩家正在通关**
   - **触发条件：** 玩家在章节末间通关瞬间 + 章节 BGM 切换瞬间重叠
   - **音频预期：** 章节 BGM 不立即切走，先播放通关音（-6dB / -3dB），通关音结束后再切下章 BGM（2s fade-in）

3. **房间 BGM 变奏失败（同 BGM 不同循环点对不上）**
   - **触发条件：** 玩家通关后从 1-2 → 1-3，BGM 循环点对不上
   - **音频预期：** BGM 重新从循环点（84s / 86s / 88s）开始播放，1s fade-in 避免断裂

4. **Boss 房 BGM 与章节 BGM 风格冲突**
   - **触发条件：** 玩家进入 3-8 Boss 房时章节 BGM 渐出 + Boss 房 BGM 渐入
   - **音频预期：** Boss 房 BGM 用**同主旋律的紧迫变奏**（非独立 BGM），3s cross-fade 避免断裂

5. **主旋律回归与章节 BGM 不衔接**
   - **触发条件：** 玩家通关瞬间 + 章节 BGM 渐出 + 主旋律回归瞬间重叠
   - **音频预期：** 主旋律回归**优先**——章节 BGM 自然结束，1s 内主旋律回归

6. **应急主题未制作但已触发**
   - **触发条件：** PlayerPrefs `emergencyTheme = "ep-1"` 但 `bgm_ep1.wav` 文件不存在
   - **音频预期：** 主旋律 BGM 持续播放 + Console 警告 "EP-1 主题未制作，使用主旋律 fallback"

7. **EP-3 "戏剧化主题" 与音频版权冲突**
   - **触发条件：** 应急使用 EP-3 但 Suno/Udio 商用协议不允许戏剧化主题
   - **音频预期：** EP-3 主题**重新生成**（5h 工时），直至协议允许
