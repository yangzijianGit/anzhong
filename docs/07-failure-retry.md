# 失败重试设计

## 7.1 无死亡无惩罚下的挑战感来源

**核心挑战来源：**

| 来源 | 实现方式 | 效果 |
|------|----------|------|
| **认知挑战** | 槽位配置的逻辑推理 | 成就感来自"想通了" |
| **空间直觉** | 想象切换后的房间结构 | 顿悟快感 |
| **最优解追求** | 设计多解法，鼓励最短步数 | 重玩动力 |
| **收集欲** | 隐藏成就/快速通关分数 | 完美主义驱动 |

## 7.2 防止"无脑乱按"的设计

虽然允许无限次切换，但通过以下设计试错成本：

| 设计 | 作用 |
|------|------|
| **观察时间成本** | 玩家需要花时间理解当前配置，然后再试 |
| **切换动画** | 0.2s动画让玩家看到每次变化，无法快速跳过 |
| **无快捷解法** | 组合空间大，穷举不现实（3槽位×3选项=27种组合） |
| **配置记忆** | 重置后忘记之前的配置，需要重新推理 |

## 7.3 重置激励机制

| 机制 | 说明 |
|------|------|
| **R键即时响应** | 无加载，房间瞬间恢复初始状态 |
| **提示系统（可选）** | 卡关太久提供渐进式hint |
| **重置计数器** | 记录本房间重置次数，完成后显示 |
| **"再来一次"成就感** | 鼓励玩家把这当成学习过程而不是失败 |

## 7.4 重置功能规格

- **R键即时响应** — 无加载，房间瞬间恢复初始状态
- **提示系统（可选）** — 卡关太久提供渐进式hint
- **重置计数器** — 记录本房间重置次数，完成后显示
- **"再来一次"成就感** — 鼓励玩家把这当成学习过程而不是失败
