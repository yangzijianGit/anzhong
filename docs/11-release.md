# 运营发布计划

## 10.1 平台选择

| 平台 | 优势 | 劣势 | 建议 |
|------|------|------|------|
| **Itch.io** | 完全免费、易上手、 Indie友好 | 流量较低 | 第一优先：免费发布 |
| **Steam** | 大流量、变现能力强 | 审核费$100、需商店页 | 第二阶段：付费发布 |
| **移动端(iOS/Android)** | 日活高、触达易 | 适配成本、买量贵 | 暂不考虑（首版） |
| **Nintendo Switch** | 独立游戏聚集地 | 审核严格、移植成本 | 后续调研 |

## 10.2 发布策略

**Itch.io阶段（推荐）：**
- 发布时间：开发完成即发布
- 价格：免费（或Pay What You Want，最低$1）
- 目的：收集反馈、建立玩家社区

**Steam阶段（进阶）：**
- 定价策略：$4.99（初版）→ $6.99（正式版）
- 首月促销：EA期间15% OFF
- 理想发布窗口：大型促销活动前后（如Summer Sale）

## 10.3 宣传节点

| 时间点 | 动作 | 内容 |
|--------|------|------|
| T-3个月 | 建立社媒 | Twitter/X账号、Discord服务器 |
| T-2个月 | 预告片 | 30秒预告片+Steam页面 |
| T-1个月 | 试玩版 | 发布Demo、邀请主播试玩 |
| T-1周 | 发力宣传 | KOL推广、新闻稿投递 |
| 发布日 | 正式发布 | 多平台发布公告 |
| T+2周 | 更新1 | 首日补丁+Bug修复 |
| T+1月 | 收集反馈 | 基于玩家反馈优化 |
